The Managing Director's Autonomous Coding Swarm (assistant-to)1. Philosophy & Architectureassistant-to is a strictly bound, multi-agent orchestrator shipped as a single compiled Go binary. It prioritizes zero context bloat and total sandboxing. Following the "Agentic Engineering" philosophy, it focuses on system-level orchestration where the developer acts as the system architect. Agents communicate via a local SQLite mailbox, work strictly in isolated Git worktrees, and are supervised by a 3-tier watchdog protocol.Tech StackLanguage: Go (1.22+)CLI / TUI: spf13/cobra (routing), charmbracelet/bubbletea (dashboard), charmbracelet/huh (interactive forms).State / IPC: modernc.org/sqlite (Pure Go SQLite, WAL mode for concurrent multi-agent access).Isolation: tmux (process persistence), git worktree (filesystem isolation).Runtime Intelligence: Pluggable AgentRuntime interface for various LLM CLIs (Claude, Gemini, etc.).2. Core WorkflowsA. Advanced Prompt Management (Composition Logic)Inspired by Canopy, the system manages prompts as versioned artifacts.Inheritance: Base prompts in .assistant-to/prompts/ define the "HOW" (workflow/safety).Overlays: Task-specific data defines the "WHAT" (scope/files).Composition: Use at prompt render to resolve fragments into a final AT_INSTRUCTIONS.md injected into the agent's worktree.B. 4-Tier Merge Resolution StrategyTo prevent bottlenecks in decentralized work, the system uses a tiered approach :Tier 1: Mechanical Merge: Standard git merge or rebase.Tier 2: Algorithmic Synthesis: Union-merge rules for structured files (JSONL/YAML) to resolve concurrent metadata updates without conflicts.Tier 3: Contextual Rebase: Attempt to rebase the feature branch onto the latest main branch to clear stale commit conflicts.Tier 4: AI-Assisted Resolution: Spawn a Merger agent with full diff context to resolve semantic conflicts before final integration.C. The 3-Tier Watchdog SystemMaintaining fleet health without manual monitoring :Tier 0 (Mechanical): Monitors PID liveness and tmux connectivity. Flags a STALL if no output is detected for 5 minutes.Tier 1 (AI Triage): Spawns a transient agent to read the last 1,000 lines of the stalled agent’s transcript. It identifies failure modes (e.g., logic loops) and sends a mail stimulus or nudges the agent via tmux send-keys.Tier 2 (Monitor Agent): A persistent agent that reviews task specifications vs. actual progress to detect "objective drift".5. Local Intelligence Layer (MCP Integration)assistant-to integrates a local code intelligence engine (similar to KotaDB) that indexes the repository into a separate SQLite database.Tooling: Exposes search_dependencies and analyze_change_impact tools via stdio transport.Impact Analysis: Before a Builder starts, it receives a map of affected files to prevent locally-optimal but globally-broken changes.6.
